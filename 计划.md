# GUI Vision Skill 开发计划

## 任务目标

创建一个 OpenCode skill，使主 Agent 能够通过视觉能力操作图形界面。

## 技术栈

- **模型**: qwen-3.5-plus (低延迟 + 强视觉能力)
- **SDK**: openai (最新版)
- **截图**: mss
- **控制**: pyautogui

## 架构设计

```
gui-vision/
├── SKILL.md              # 触发指导和工作流程
├── scripts/
│   ├── __init__.py       # 模块导出
│   ├── agent.py          # 视觉 Agent 主循环
│   ├── vision.py         # 截图和图像处理
│   ├── action.py         # GUI 动作执行
│   └── utils.py          # 坐标转换等工具
├── config/
│   └── config.yaml.example
└── requirements.txt
```

## 核心流程

```
Observe → Think → Act → Verify
   ↓         ↓       ↓       ↓
 截图    模型决策   执行操作   验证结果
```

### 1. Observe (观察)
- 使用 mss 截取当前屏幕
- 转换为 base64 发送给模型

### 2. Think (思考)
- 调用 qwen-3.5-plus 视觉接口
- 传入截图 + 用户指令 + 操作历史
- 返回操作类型和坐标

### 3. Act (执行)
- 支持的操作: click, double_click, right_click, type, drag, scroll, wait
- 坐标映射: 1000×1000 归一化 → 实际像素

### 4. Verify (验证)
- 检查操作是否成功
- 决定是否继续循环

## 坐标映射

模型返回 `[y, x]` 归一化坐标 (0-1000)，需要转换:

```python
pixel_x = norm_x * screen_width / 1000
pixel_y = norm_y * screen_height / 1000
```

## 安全机制

| 操作类型 | 风险级别 | 确认策略 |
|---------|---------|---------|
| click, type, scroll | 低 | 自动执行 |
| double_click, drag | 中 | 首次确认 |
| delete, close | 高 | 每次确认 |

## 状态管理

- 操作历史记录 (最近5步)
- 防止无限循环 (最大50步)
- 支持任务中断和重置

## 配置

```yaml
api:
  api_key: "your-api-key"
  base_url: "https://dashscope.aliyuncs.com/compatible-mode/v1"
  model: "qwen-3.5-plus"

screen:
  width: 1920
  height: 1080

safety:
  auto_confirm_low_risk: true
  require_confirmation: ["delete", "close", "shutdown"]

timing:
  action_delay: 0.5
  screenshot_delay: 1.0
```

## 使用方式

```python
from scripts.agent import VisionAgent

agent = VisionAgent()
agent.execute("点击屏幕上的'保存'按钮")
```
